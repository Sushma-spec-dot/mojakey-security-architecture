# Example Permissions
Parent or guardian: manage own family profile, view own children, enroll own children, make payments

Institution staff: check students in or out, view institution attendance records, manage operational workflows

Coach: view assigned class rosters, mark attendance where allowed

Institution administrator: manage staff roles within the institution, review attendance and program data, handle institution-level settings

Platform super admin: platform-wide administration, support, audit review, restricted high-privilege actions

# Why RBAC Matters

RBAC matters because Mojakey involves child-related workflows, family data, institution operations, and payment-related actions. Users should only access the minimum data and actions required for their role, and every role must remain scoped to the correct institution or family context.
